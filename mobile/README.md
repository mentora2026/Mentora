# Mobile App — Adaptive Psychological Monitoring and Support Platform

Step 4 deliverable: Flutter patient-facing mobile application implementing
the Mobile App Structure defined in Step 1 (Section 10), fully in Arabic
with RTL layout, consuming the Step 2/3 FastAPI backend.

## 1. Tech Stack

- **Framework**: Flutter (Dart >= 3.6)
- **State management**: `provider`
- **Networking**: `http` with a custom `ApiClient` (JWT auth + automatic
  refresh-token retry on 401)
- **Secure storage**: `flutter_secure_storage` (JWT tokens)
- **Charts**: `fl_chart` (mood trend line chart, risk progression bar chart)
- **Localization**: Arabic-only (`ar`), RTL via `Directionality`

## 3. Setup

### Prerequisites

- Flutter SDK (stable channel, Dart >= 3.6) - install from
  `https://docs.flutter.dev/get-started/install`
- A running instance of the Step 2/3 backend (see `backend/README.md`)

### Generate platform projects

This repository ships only the cross-platform `lib/` source and
`pubspec.yaml`. Generate the Android/iOS/etc. platform folders with:

```bash
cd mobile
flutter create .
```

This populates `android/`, `ios/`, and other platform directories using the
`name`/`description` already defined in `pubspec.yaml`, without overwriting
`lib/`.

> **Android note**: `flutter_secure_storage` requires `minSdkVersion >= 18`
> in `android/app/build.gradle` (the default generated by recent Flutter
> versions already satisfies this).

### Install dependencies

```bash
flutter pub get
```

### Configure the backend URL

Edit `lib/core/constants/api_constants.dart`:

```dart
static const String baseUrl = "http://10.0.2.2:8000/api/v1";
```

- `10.0.2.2` is the Android emulator's alias for the host machine's
  `localhost` - use this if running the backend locally with `docker compose up`.
- For a physical device, replace with your machine's LAN IP
  (e.g., `http://192.168.1.50:8000/api/v1`).
- For a deployed backend, use its public URL
  (e.g., `https://api.yourapp.com/api/v1`).

### Run

```bash
flutter run
```

## 4. App Flow

1. **AuthGate** (`presentation/shared/auth_gate.dart`) checks for a stored
   JWT. If absent → **Login/Register**.
2. After login, **AuthGate** checks `patient_profiles.onboarding_completed`.
   If `false` → **Profile Setup** (chronic conditions, disease duration,
   lifestyle info). On save, the user proceeds to the main app.
3. **MainShell** provides bottom-navigation across:
   - **الرئيسية (Home)**: latest risk level + explanation, "Start Interview"
     entry point, quick mood log shortcut.
   - **التوصيات (Recommendations)**: personalized, expandable cards with
     helpful/not-helpful feedback.
   - **التقارير (Reports)**: weekly Arabic summary, mood trend line chart,
     risk progression bar chart (both via `fl_chart`).
   - **الإشعارات (Notifications)**: list with read/unread state.
   - **حسابي (Profile)**: edit profile, logout.

## 5. Adaptive Interview Chat (`presentation/chat/`)

- On open, resumes any in-progress session or starts a new one
  (`POST /interviews/start`).
- Renders the full conversation as chat bubbles (bot messages right-aligned
  in a light bubble, patient messages left-aligned in a colored bubble -
  RTL-correct since Arabic reads right-to-left).
- `ChatInputBar` provides a free-text field plus an optional **1-5 scale
  picker** (toggled via an icon) for `scale_1_5`-type questions.
- Each submitted answer calls `POST /interviews/{id}/answer`; the bot's next
  message (or closing message) is appended automatically.
- A **typing indicator** is shown while waiting for the backend response
  (which may include sentiment analysis + LLM rephrasing on the backend side).
- The patient can end the session early (with confirmation) via
  `POST /interviews/{id}/end`.
- When a session ends (including a crisis-language early termination handled
  by the Step 3 AI engine), a "Start New Session" button appears.

## 6. Fonts (optional)

The app currently uses the platform default font, which renders Arabic
correctly. To use a custom Arabic font (e.g., Cairo or Tajawal):

1. Download the font `.ttf` files into `assets/fonts/`.
2. Uncomment the `fonts:` section in `pubspec.yaml`.
3. Uncomment `fontFamily: 'Cairo'` in `lib/core/theme/app_theme.dart`.

## 7. Notes on FCM (Push Notifications)

`POST /devices/register` is wired up in `NotificationRepository.registerDevice()`
but not yet called automatically. To complete FCM integration:

1. Add `firebase_core` and `firebase_messaging` to `pubspec.yaml`.
2. Configure Firebase for Android/iOS (`flutterfire configure`).
3. On app startup (after login), request notification permissions, obtain the
   FCM token, and call `registerDevice(token)`.
